# CV3.E2 — Inventory Optimization Engine

**CV:** [CV3 — Explainable Inventory Recommendations](../index.md)
**Status:** ⚪ Planned
**Depends on:** CV3.E1
**Prerequisite for:** CV3.E3

---

## What This Is

Operations Research, not LLMs. Pyomo / OR-Tools for the optimization itself, Monte Carlo for stockout probability estimation, multi-objective Pareto for the cost ↔ risk trade-off. Recommendations are *generated by deterministic code* and *explained by deterministic code*.

---

## Stories

| Code | Story | Status |
|------|-------|--------|
| CV3.E2.S1 | Pyomo-based min/max + safety stock + reorder point optimization | 📥 Backlog |
| CV3.E2.S2 | Monte Carlo stockout-probability estimator with lead-time distribution | 📥 Backlog |
| CV3.E2.S3 | Multi-objective Pareto frontier (capital × risk × service level) | 📥 Backlog |
| CV3.E2.S4 | Action recommendation logic (buy / transfer / excess reduction) | 📥 Backlog |
| CV3.E2.S5 | Cross-site transfer optimizer (network-wide LP) | 📥 Backlog |
| CV3.E2.S6 | Engine versioned in `ml.model_registry`; deterministic given seed | 📥 Backlog |

---

## Done Condition

- The engine produces min/max + reorder point + recommended action for every active item.
- Stockout probability claims are reproducible from a fixed seed + lead-time distribution.
- The Pareto frontier is computable and queryable.
- Recommended transfers respect cross-site service-level constraints.

---

## Out of Scope
